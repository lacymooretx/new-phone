Audio Prompts

Upload, manage, and organize audio files used for music on hold, greetings, IVR prompts, announcements, voicemail, and notifications.

Overview

Audio prompts are pre-recorded audio files used throughout the phone system. They provide the voice of your organization — from the greeting callers hear when they first connect, to the music they hear while on hold, to the announcements played in IVR menus and queue wait messages.

The platform stores all audio files in S3-compatible object storage (MinIO or AWS S3), organized by tenant. Each prompt tracks metadata including format, duration, sample rate, and a SHA-256 hash for integrity verification. Playback is handled through time-limited signed URLs for secure access.

Tenant Isolation
Audio prompts are scoped to individual tenants. Each tenant manages their own library of prompts, and prompts from one tenant are never accessible to another. System-wide default prompts are provided by the platform for common use cases.

Uploading Audio Prompts

Navigate to System > Audio Prompts and click Upload Prompt. Provide a name, select a category, and choose the audio file from your computer. The platform validates the file format and extracts metadata (duration, sample rate, file size) automatically upon upload.

The upload process involves two steps:

  1. Create the prompt record — Submit the name, description, and category to create the metadata entry
  2. Upload the audio file — Attach the actual audio file to the created prompt record

After upload, the file is stored in the configured S3/MinIO bucket and the prompt record is updated with the storage path, file size, duration, format details, and integrity hash.

Field Reference

AudioPromptCreate Fields

Field Type Required Description

description string No Optional description of the prompt's content or purpose

The full response includes all create fields plus system-managed and file metadata fields:

Field Type Description

Returned when requesting a playback URL for a prompt:

Field Type Description
url string Pre-signed URL for streaming or downloading the audio file
expires_in_seconds integer Seconds until the signed URL expires. Default: 300 (5 minutes)

Categories

Categories help organize your prompt library and make it easier to find the right audio file when configuring system features. Each prompt is assigned to exactly one category.

Category Purpose Typical Use
GENERAL Default category for uncategorized prompts Miscellaneous audio files, system sounds
MOH Music on Hold Background music played while callers wait on hold or in queues
GREETING Welcome greetings IVR welcome messages, auto-attendant greetings, business-hours/after-hours greetings
ANNOUNCEMENT Informational announcements Queue position updates, holiday closure notices, service alerts
VOICEMAIL Voicemail prompts Voicemail greeting ("Please leave a message"), unavailable messages, busy messages
NOTIFICATION System notifications Tone alerts, transfer notifications, recording warnings ("This call may be recorded")

Audio Format Requirements

The platform accepts WAV and MP3 audio files. For optimal telephony quality and compatibility, WAV files at 8 kHz or 16 kHz are recommended.

Property WAV (Recommended) MP3
Format PCM WAV (RIFF) MPEG Audio Layer 3
Sample rates 8000, 16000, 32000, 44100, 48000 Hz 8000, 16000, 22050, 44100, 48000 Hz
Channels Mono (1 channel) preferred Mono or stereo (downmixed to mono)
Bit depth 16-bit N/A (lossy compression)
Recommended 8 kHz / 16-bit / Mono for telephony Acceptable for MOH with higher quality needs
Automatic Transcoding
FreeSWITCH handles real-time transcoding between formats and sample rates as needed. However, uploading files in the native telephony format (8 kHz WAV) reduces processing overhead and ensures the best quality without transcoding artifacts.
File Size Limits
Audio files are subject to upload size limits configured by your administrator. For long-duration music on hold, consider using multiple shorter tracks in a playlist rather than a single large file. As a guideline, 1 minute of 8 kHz mono WAV audio is approximately 960 KB.

Storage

Audio files are stored in S3-compatible object storage (MinIO for self-hosted deployments, AWS S3 for cloud deployments). Each file is stored under a tenant-specific path to ensure isolation.

The storage path follows this structure:

{bucket}/{tenant_id}/audio-prompts/{prompt_id}/{filename}

Key storage characteristics:

  • Durability: Object storage provides high durability with redundant copies
  • Integrity: SHA-256 hash is computed at upload and can be verified at any time
  • Caching: Frequently-used prompts may be cached locally on the FreeSWITCH server for low-latency playback
  • Cleanup: Deleting a prompt record also removes the file from object storage

Playback and Signed URLs

Audio prompts are not served directly from object storage. Instead, the API generates pre-signed URLs with a configurable expiration time (default: 5 minutes / 300 seconds). This ensures that audio files are only accessible to authenticated users for a limited time window.

Use the playback endpoint to get a signed URL:

GET /api/v1/audio-prompts/{id}/playback

Response:
{
  "url": "https://minio.example.com/audio/tenant-123/audio-prompts/prompt-456/greeting.wav?X-Amz-...",
  "expires_in_seconds": 300
}

The signed URL can be used in the browser's audio player for preview, or passed to external applications. After the expiration time, the URL becomes invalid and a new one must be generated.

Where Prompts Are Used

Audio prompts are referenced throughout the system wherever audio playback is needed. The following features support prompt assignment:

Feature Prompt Type Description
IVR Menus GREETING Welcome message and menu option prompts
Queues MOH, ANNOUNCEMENT Hold music and periodic position/wait-time announcements
Conferences MOH, NOTIFICATION Waiting room music and join/leave notification tones
Voicemail VOICEMAIL Personalized unavailable, busy, and temporary greetings
Ring Groups MOH Music or ringing tone while the group rings
Extensions MOH Per-extension hold music
Inbound Routes ANNOUNCEMENT Pre-answer announcements before routing

Text-to-Speech (TTS) Generation Coming Soon

A future release will add the ability to generate audio prompts from text using text-to-speech engines. This will allow administrators to type a greeting or announcement and have the system generate a professional-sounding audio file automatically.

Planned TTS features include:

  • Multiple voice options (male/female, different accents)
  • Multiple language support
  • SSML support for fine-tuned pronunciation and pacing
  • Preview before saving
  • Provider options including cloud TTS services (Google, AWS, ElevenLabs)
TTS Not Yet Available
Text-to-speech prompt generation is planned for a future release. Currently, all prompts must be uploaded as pre-recorded audio files.

Best Practices

Audio Quality

  • Record in a quiet environment with minimal background noise
  • Use a quality microphone; avoid laptop or phone microphones for professional prompts
  • Speak clearly and at a moderate pace
  • Normalize audio levels to prevent volume jumps between prompts
  • Leave a brief silence (0.5–1 second) at the beginning and end of each recording

File Management

  • Use descriptive names that indicate content and context (e.g., "Main IVR - Business Hours Greeting" not "greeting1")
  • Assign appropriate categories to keep the library organized
  • Use the description field to note when the prompt was recorded and by whom
  • Regularly review and remove outdated prompts (seasonal greetings, old announcements)
  • Keep backup copies of important recordings outside the system

File Size and Duration

  • Use 8 kHz mono WAV for telephony prompts — smallest size, best compatibility
  • Keep greetings and announcements concise (under 30 seconds when possible)
  • For MOH, use MP3 if higher audio quality is desired for music tracks
  • Avoid uploading unnecessarily high sample rates (48 kHz) for telephony use — the extra quality is lost in phone audio
