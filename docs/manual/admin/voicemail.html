Voicemail Boxes & Messages

Create and manage voicemail boxes, configure greetings and notifications, and work with voicemail messages.

Overview

Voicemail boxes provide a destination for unanswered calls. When a call reaches an extension that is busy, unanswered, or has no registered device and no call forwarding destination is configured, the caller hears a greeting and can leave a recorded message.

Each voicemail box is an independent entity identified by a mailbox_number. Boxes are linked to extensions via the extension's voicemail_box_id field, but a box can exist without being linked (e.g., a shared departmental mailbox or a pre-provisioned box awaiting assignment).

Users access their voicemail by dialing a feature code from their phone, through the web UI voicemail panel, or via the mobile app. Administrators can manage all voicemail boxes within their tenant and access message metadata (but message playback respects role-based access controls).

Voicemail Boxes vs. Extensions
A voicemail box is separate from an extension. Create the voicemail box first, then link it to an extension by setting the extension's voicemail_box_id. One extension can have one voicemail box, and one voicemail box should be linked to one extension (though a box can technically be shared by updating multiple extensions to point to the same box).

Creating a Voicemail Box

To create a new voicemail box, submit a POST request to /api/v1/voicemail/boxes with a JSON body matching the VoicemailBoxCreate schema, or use the admin UI:

  1. Navigate to Reports > Voicemail in the admin interface.
  2. Click Add Voicemail Box.
  3. Enter the mailbox number (e.g., "100"). This typically matches the associated extension number for user convenience.
  4. Set the PIN (4–20 digits). The user will need this to check voicemail from a phone.
  5. Choose a greeting type and configure email notification settings.
  6. Click Save.

After creating the box, link it to an extension by editing the extension and setting its voicemail_box_id to the new box's id.

VoicemailBoxCreate Field Reference

Field Type Required Constraints Description
mailbox_number string Yes 1–20 characters The voicemail box number, used for access from phones. Typically matches the extension number (e.g., "100").
pin string Yes 4–20 characters Numeric PIN for phone-based voicemail access. Users enter this when dialing into the voicemail system.
greeting_type enum No Default: DEFAULT Which greeting callers hear. One of: DEFAULT, CUSTOM, COMPANY. See Greeting Types.
email_notification boolean No Default: true When enabled, the system sends an email notification each time a new voicemail message is left.
notification_email string No Max 320 characters Email address for voicemail notifications. If not set and email_notification is enabled, the system uses the linked user's email address.
max_messages integer No 1–9999, default 100 Maximum number of messages the box can store. When the limit is reached, new callers hear a "mailbox full" announcement.

VoicemailBoxResponse Fields

API responses for voicemail boxes include all create fields plus the following system-managed fields:

Field Type Description
id UUID Unique identifier for the voicemail box.
tenant_id UUID The tenant this voicemail box belongs to. Set automatically; enforced by row-level security.
mailbox_number string The voicemail box number.
greeting_type string Current greeting type (DEFAULT, CUSTOM, or COMPANY).
email_notification boolean Whether email notifications are enabled.
notification_email string | null Override email address for notifications, or null if using the linked user's email.
max_messages integer Maximum message capacity.
is_active boolean true if the voicemail box is active. Inactive boxes reject new messages.
created_at datetime Timestamp when the box was created (UTC).
updated_at datetime Timestamp of the last modification (UTC).

Greeting Types

The greeting_type field determines which greeting a caller hears before leaving a message:

Type Behavior
DEFAULT The system plays a generic greeting: "The person at extension [number] is not available. Please leave a message after the tone." No custom recording needed.
CUSTOM The system plays a user-recorded greeting. The user records this by dialing into the voicemail system and following the greeting management prompts, or by uploading an audio file through the web UI.
COMPANY The system plays the tenant-wide company greeting. This is a single recording shared by all voicemail boxes in the tenant that use this type. Managed under Audio Prompts.

Users can switch between greeting types from their phone or the web UI without losing previously recorded custom greetings. The custom recording is preserved even when the box is set to DEFAULT or COMPANY.

Email Notifications

When email_notification is enabled (the default), the platform sends an email each time a new voicemail message is deposited in the box. The notification includes:

  • Caller name and number (if available from caller ID)
  • Date and time of the message
  • Duration of the recording
  • A link to play the message in the web UI (requires authentication)

The notification is sent to the address in notification_email. If that field is empty, the system falls back to the email address of the user linked to the extension that is linked to this voicemail box. If no email can be determined, the notification is silently skipped.

Email Delivery
Voicemail notification emails are sent via the platform's configured SMTP provider. Ensure your tenant's email settings are configured correctly under System > Admin Ops. Emails may be subject to spam filtering by the recipient's mail provider.

Managing Voicemail Messages

Voicemail messages are stored as audio files in S3/MinIO object storage. The API provides endpoints to list, filter, play, update, forward, and delete messages.

VoicemailMessageResponse Fields

Field Type Description
id UUID Unique identifier for the message.
tenant_id UUID Tenant that owns this message.
voicemail_box_id UUID The voicemail box this message belongs to.
caller_number string Phone number of the caller who left the message.
caller_name string Caller ID name of the caller (if available).
duration_seconds integer Length of the voicemail recording in seconds.
storage_path string Object storage path to the audio file (internal reference).
storage_bucket string S3/MinIO bucket name where the file is stored.
file_size_bytes integer Size of the audio file in bytes.
format string Audio format of the recording: "wav" or "mp3".
sha256_hash string SHA-256 hash of the audio file for integrity verification.
is_read boolean true if the message has been listened to or marked as read.
is_urgent boolean true if the caller flagged the message as urgent.
folder enum Current folder: INBOX, OLD, WORK, FAMILY, or FRIENDS.
call_id string The call ID from the originating call, for correlation with CDR records.
email_sent boolean true if a notification email was sent for this message.
is_active boolean true if the message is active (not soft-deleted).
created_at datetime Timestamp when the message was recorded (UTC).
updated_at datetime Timestamp of the last modification (UTC).

Updating Messages

Use PATCH /api/v1/voicemail/messages/{id} with a VoicemailMessageUpdate body to update message metadata:

Field Type Description
is_read boolean Mark the message as read or unread.
folder enum Move the message to a different folder (INBOX, OLD, WORK, FAMILY, FRIENDS).

Filtering Messages

Use GET /api/v1/voicemail/boxes/{box_id}/messages with query parameters to filter and paginate messages:

Parameter Type Description
folder enum Filter by folder (INBOX, OLD, WORK, FAMILY, FRIENDS).
is_read boolean Filter by read status (true for read, false for unread).
date_from datetime Return messages on or after this date (UTC).
date_to datetime Return messages on or before this date (UTC).
limit integer Maximum results to return (1–1000, default 50).
offset integer Number of results to skip (default 0). Use with limit for pagination.

Forwarding Messages

Forward a voicemail message to another voicemail box using POST /api/v1/voicemail/messages/{id}/forward with a VoicemailForwardRequest body:

Field Type Required Description
target_box_id UUID Yes The ID of the voicemail box to forward the message to. Must be a box within the same tenant.

Forwarding creates a copy of the message in the target box. The original message remains in the source box. The forwarded copy appears as a new unread message in the target box's INBOX folder.

Message Summary

Get a quick count of unread messages using GET /api/v1/voicemail/boxes/{box_id}/summary. Returns a VoicemailMessageSummaryResponse:

Field Type Description
voicemail_box_id UUID The voicemail box ID.
mailbox_number string The mailbox number for display purposes.
unread_count integer Number of unread messages in the box.

Message Folders

Voicemail messages are organized into folders. Users can move messages between folders for organization. The available folders are:

Folder Description
INBOX Default folder for all new messages. New voicemails arrive here.
OLD Messages that have been listened to. Some phone interfaces automatically move messages here after playback.
WORK User-defined organizational folder for work-related messages.
FAMILY User-defined organizational folder for family messages.
FRIENDS User-defined organizational folder for messages from friends.

Messages are moved between folders using the VoicemailMessageUpdate endpoint. The web UI and mobile app provide drag-and-drop folder management. Phone-based voicemail access supports folder navigation through the IVR menu prompts.

New Voicemail Arrives └─ Deposited in INBOX (is_read: false) ├─ User listens → is_read: true │ ├─ Auto-move to OLD (phone behavior) │ └─ Or keep in INBOX (web/mobile behavior) ├─ User moves to WORK / FAMILY / FRIENDS ├─ User forwards to another box → Copy in target INBOX └─ User deletes → Soft-delete (is_active: false)

Message Playback

To play a voicemail message through the web UI or mobile app, the client requests a temporary signed URL from GET /api/v1/voicemail/messages/{id}/playback. The response is a VoicemailMessagePlaybackResponse:

Field Type Description
url string Signed URL for direct audio file access. This URL authenticates the request to S3/MinIO without requiring additional credentials.
expires_in_seconds integer Number of seconds until the URL expires. Default is 300 (5 minutes). After expiry, a new URL must be requested.
Signed URL Security
Signed URLs provide temporary, unauthenticated access to the audio file. They should not be shared or bookmarked. The 5-minute expiry limits the window of exposure. If a user needs to listen again after the URL expires, they simply request a new one.

Playback is also available from physical phones by dialing the voicemail feature code and navigating the IVR prompts. Phone-based playback does not use signed URLs; the audio is streamed directly through the RTP media path.

PIN Management

The voicemail PIN is required when accessing voicemail from a phone. It is set during box creation and can be reset at any time by an administrator.

To reset the PIN, call POST /api/v1/voicemail/boxes/{id}/reset-pin. The response returns a PinResetResponse:

Field Type Description
pin string The new PIN. Returned only in this response. Communicate it to the user securely.

The PIN is stored as a hashed value; it cannot be retrieved after the reset response. If a user forgets their PIN, the administrator must reset it and provide the new PIN. Users can also change their own PIN from the voicemail IVR menu on their phone (option typically available under "mailbox options").

PIN Best Practices
Encourage users to change the default PIN immediately after receiving it. PINs should be at least 6 digits and should not be easily guessable sequences (e.g., "1234", "0000", or the extension number). The platform enforces a minimum of 4 digits and a maximum of 20.

Storage (S3/MinIO)

Voicemail recordings are stored in S3-compatible object storage (MinIO in self-hosted deployments, or AWS S3 / any S3-compatible service). Each tenant's messages are stored in an isolated path structure.

Audio Format

The platform stores voicemail in two formats:

  • WAV — Uncompressed audio. Higher quality and larger file size. Default recording format from FreeSWITCH. Typical size: ~1 MB per minute at 16-bit 8kHz mono.
  • MP3 — Compressed audio. Smaller file size suitable for email attachments and mobile playback. Typical size: ~120 KB per minute.

The format field on each message indicates which format is stored. The platform can be configured to store both formats or to transcode on the fly.

File Integrity

Each voicemail message includes a sha256_hash field containing the SHA-256 hash of the audio file at the time of recording. This can be used to verify that the file has not been modified in storage, which is important for compliance and legal discovery scenarios.

Storage Path Structure

{bucket}/
  {tenant_id}/
    voicemail/
      {voicemail_box_id}/
        {message_id}.{format}

The storage_bucket and storage_path fields on each message provide the exact location. Direct access to storage requires S3 credentials and is typically only used for backup or migration purposes.

Linking to Extensions

Voicemail boxes are connected to extensions through the extension's voicemail_box_id field. To link a voicemail box to an extension:

  1. Note the voicemail box id (UUID) from the box list or creation response.
  2. Edit the extension and set voicemail_box_id to that UUID.
  3. Save the extension. Unanswered calls to that extension will now route to voicemail when no call forwarding destination handles the call.

To unlink a voicemail box from an extension, set the extension's voicemail_box_id to null. The voicemail box and its messages are preserved; only the routing link is removed.

Matching Numbers Convention
While not required, it is a common convention to give the voicemail box the same mailbox_number as the extension's extension_number. This makes it intuitive for users when accessing voicemail from their phone — they dial into the system and the mailbox number matches their extension.

API Examples

Create a Voicemail Box

POST /api/v1/voicemail/boxes
Content-Type: application/json
Authorization: Bearer <token>

{
  "mailbox_number": "100",
  "pin": "839472",
  "greeting_type": "DEFAULT",
  "email_notification": true,
  "notification_email": "jane.doe@acmecorp.com",
  "max_messages": 200
}

List Unread Messages

GET /api/v1/voicemail/boxes/{box_id}/messages?is_read=false&folder=INBOX&limit=25
Authorization: Bearer <token>

Mark Message as Read

PATCH /api/v1/voicemail/messages/{message_id}
Content-Type: application/json
Authorization: Bearer <token>

{
  "is_read": true
}

Move Message to WORK Folder

PATCH /api/v1/voicemail/messages/{message_id}
Content-Type: application/json
Authorization: Bearer <token>

{
  "folder": "WORK"
}

Forward a Message

POST /api/v1/voicemail/messages/{message_id}/forward
Content-Type: application/json
Authorization: Bearer <token>

{
  "target_box_id": "b2c3d4e5-f6a7-8901-bcde-f23456789012"
}

Get Playback URL

GET /api/v1/voicemail/messages/{message_id}/playback
Authorization: Bearer <token>

Response:
{
  "url": "https://minio.example.com/voicemail/tenant-id/.../msg.wav?X-Amz-...",
  "expires_in_seconds": 300
}

Reset PIN

POST /api/v1/voicemail/boxes/{box_id}/reset-pin
Authorization: Bearer <token>

Response:
{
  "pin": "742918"
}
