Holiday Calendars

Centrally manage holidays and special closures that automatically override time condition routing.

Overview

Holiday calendars provide a centralized way to define dates when your organization is closed or operating on a reduced schedule. When a holiday calendar is linked to a time condition, any holiday entry that covers the current date and time will override the normal rules, forcing the time condition to evaluate as "no-match" (after-hours routing).

This eliminates the need to manually update time conditions before each holiday. Define your holidays once in a calendar, link it to your time conditions, and the system handles the rest automatically.

One Calendar, Many Time Conditions
A single holiday calendar can be linked to multiple time conditions. Create one company-wide calendar and reference it from every time condition that should observe those holidays.

Creating Holiday Calendars

Navigate to System > Holiday Calendars and click Add Calendar. Provide a name for the calendar and add one or more holiday entries. Each entry represents a single holiday or closure date.

After creating the calendar, link it to one or more time conditions by setting the holiday_calendar_id field on the time condition. Entries can be added, modified, or removed from the calendar at any time without updating the linked time conditions.

Field Reference

HolidayCalendarCreate Fields

Field Type Required Description
name string Yes Calendar name (1–100 characters). Example: "US Federal Holidays"
description string No Optional description of the calendar's purpose
entries list[HolidayEntryData] No List of holiday entries. Can be empty initially and populated later

HolidayEntryData Fields

Field Type Required Description
name string Yes Name of the holiday (1–100 characters). Example: "Christmas Day"
date date Yes The holiday date in YYYY-MM-DD format
recur_annually boolean No Repeat this entry every year on the same month/day. Default: false
all_day boolean No Whether the holiday covers the entire day. Default: true
start_time time No Start time for partial-day holidays. Format: HH:MM:SS. Only used when all_day is false
end_time time No End time for partial-day holidays. Format: HH:MM:SS. Only used when all_day is false

Response Fields

The response includes all create fields plus the following system-managed fields:

Field Type Description
id UUID Unique identifier for the calendar
tenant_id UUID Owning tenant (set automatically from auth context)
is_active boolean Whether the calendar is currently active
created_at datetime Timestamp when the calendar was created
updated_at datetime Timestamp of the last modification

Each entry in the response also includes:

Field Type Description
id UUID Unique identifier for the entry
calendar_id UUID Parent calendar reference

Recurring vs One-Time Holidays

Holiday entries can be either one-time or annually recurring, controlled by the recur_annually field.

Recurring Holidays

When recur_annually is true, the entry matches on the same month and day every year. The year portion of the date field is used only for the initial occurrence and is ignored in subsequent years.

This is ideal for fixed-date holidays like New Year's Day (January 1), Independence Day (July 4), and Christmas Day (December 25).

Floating Holidays
Holidays that fall on a different date each year (e.g., Thanksgiving, Memorial Day, Labor Day) cannot use recur_annually because they are not fixed to a specific month/day. These must be updated annually with the correct date, or you can add entries for multiple future years.

One-Time Holidays

When recur_annually is false (the default), the entry matches only on the exact date specified. This is useful for:

  • Floating holidays (Thanksgiving, Memorial Day, Easter)
  • Company-specific events (annual retreat, office move day)
  • One-off closures (emergency weather, building maintenance)

Partial-Day Holidays

Not all holidays require a full-day closure. The platform supports partial-day holidays for situations like early closures before a holiday or reduced hours on specific days.

To create a partial-day holiday, set all_day to false and provide start_time and end_time values. The holiday override only applies during the specified time window.

Example: Christmas Eve Early Close

{
  "name": "Christmas Eve Early Close",
  "date": "2026-12-24",
  "recur_annually": true,
  "all_day": false,
  "start_time": "12:00:00",
  "end_time": "23:59:59"
}

This entry causes the linked time condition to route calls to the after-hours destination from noon onward on December 24th, while the morning hours operate normally.

Example: Day After Thanksgiving (half day)

{
  "name": "Day After Thanksgiving - Half Day",
  "date": "2026-11-27",
  "recur_annually": false,
  "all_day": false,
  "start_time": "13:00:00",
  "end_time": "23:59:59"
}

Linking to Time Conditions

A holiday calendar is linked to a time condition by setting the holiday_calendar_id field when creating or updating the time condition. Once linked, the time condition automatically checks the calendar before evaluating its rules.

The linking relationship is one-to-many: a single calendar can be referenced by many time conditions, but each time condition can only reference one calendar. If you need different sets of holidays for different departments, create separate calendars.

Evaluation Order
When a time condition has a linked holiday calendar, the evaluation order is: (1) check manual override, (2) check holiday calendar, (3) evaluate time rules. A holiday match short-circuits the process and immediately routes to the no-match destination.

US Federal Holidays Example

The following example shows a complete calendar with common US holidays. Fixed-date holidays use recur_annually: true, while floating holidays require annual date updates. 

{
  "name": "US Federal Holidays 2026",
  "description": "Standard US federal holidays for 2026",
  "entries": [
    { "name": "New Year's Day",      "date": "2026-01-01", "recur_annually": true  },
    { "name": "MLK Jr. Day",         "date": "2026-01-19", "recur_annually": false },
    { "name": "Presidents' Day",     "date": "2026-02-16", "recur_annually": false },
    { "name": "Memorial Day",        "date": "2026-05-25", "recur_annually": false },
    { "name": "Independence Day",    "date": "2026-07-04", "recur_annually": true  },
    { "name": "Labor Day",           "date": "2026-09-07", "recur_annually": false },
    { "name": "Columbus Day",        "date": "2026-10-12", "recur_annually": false },
    { "name": "Veterans Day",        "date": "2026-11-11", "recur_annually": true  },
    { "name": "Thanksgiving",        "date": "2026-11-26", "recur_annually": false },
    { "name": "Day After Thanksgiving", "date": "2026-11-27", "recur_annually": false },
    { "name": "Christmas Eve",       "date": "2026-12-24", "recur_annually": true,
      "all_day": false, "start_time": "12:00:00", "end_time": "23:59:59" },
    { "name": "Christmas Day",       "date": "2026-12-25", "recur_annually": true  }
  ]
}
Annual Maintenance Required
Floating holidays (MLK Day, Presidents' Day, Memorial Day, Labor Day, Columbus Day, Thanksgiving) change dates each year. Review and update these entries annually, ideally in December for the upcoming year.

Multiple Calendar Use Cases

Organizations often benefit from maintaining multiple holiday calendars for different purposes:

Company-Wide Calendar

A primary calendar containing all company-observed holidays. Link this to time conditions for the main office, reception, and general departments.

Regional Calendars

For organizations with offices in multiple regions or countries, create separate calendars for region-specific holidays. A UK office might observe Bank Holidays, while a Canadian office observes Boxing Day and Canada Day.

Department-Specific Calendars

Some departments may have additional closure days (e.g., accounting closes for year-end, IT closes for maintenance windows). Create department-specific calendars and link them to the relevant time conditions.

Emergency / Ad-Hoc Calendar

Maintain a separate calendar for unplanned closures (weather, power outages, emergencies). This keeps your standard holiday calendar clean while providing a quick way to add temporary closures. Alternatively, use the manual override feature on the time condition for immediate, temporary routing changes.
