Ring Groups

Ring multiple extensions simultaneously, sequentially, or in round-robin order so the first available person answers the call.

Overview

A ring group is a collection of extensions that ring together when the group is called. Unlike queues, ring groups do not place callers on hold with music and announcements. They simply ring the configured extensions using the chosen strategy and, if nobody answers within the ring time, send the call to a failover destination.

Ring groups are ideal for small teams where any member can handle the call, and formal call center features (agent login, statistics, tiered routing) are not needed. Each ring group has its own dial number, so it can be called directly from other extensions or assigned as the destination of an inbound route or IVR menu option.

Ring Group vs Queue
Use a ring group for informal "ring everyone" scenarios (front desk, small sales team). Use a queue when you need hold music, position announcements, agent status tracking, and reporting.

Ring Strategies

The ring strategy determines the order and manner in which member extensions are rung. Choose the strategy that best fits how your team operates.

Simultaneous

All member extensions ring at the same time. The first person to pick up gets the call; all other phones stop ringing. This is the default strategy and the most common choice.

Incoming call → Ring Group (SIMULTANEOUS) │ ├─ Ext 101 ☎ RING ├─ Ext 102 ☎ RING → First to answer gets the call ├─ Ext 103 ☎ RING │ └─ ring_time expires (25s) → Failover destination

Sequential

Extensions are rung one at a time in member order. Each extension rings for ring_time_per_member seconds before the next one is tried. If busy members are skipped (skip_busy: true), the system moves to the next available member immediately.

Incoming call → Ring Group (SEQUENTIAL) │ ├─ Ext 101 ☎ RING (15s) → no answer ├─ Ext 102 ☎ BUSY → skip (skip_busy: true) ├─ Ext 103 ☎ RING (15s) → no answer │ └─ All members tried → Failover destination

Round Robin

Similar to sequential, but the starting member rotates with each call. The first call starts with member 1, the second call starts with member 2, and so on. This distributes calls evenly across the team over time.

Call #1 → Start at Ext 101 → 101, 102, 103 Call #2 → Start at Ext 102 → 102, 103, 101 Call #3 → Start at Ext 103 → 103, 101, 102 Call #4 → Start at Ext 101 → 101, 102, 103 (cycle repeats)

Creating a Ring Group

To create a ring group, submit a POST request with the group configuration and an initial list of member extension IDs. 

POST /api/v1/ring-groups

{
  "group_number": "600",
  "name": "Front Desk",
  "ring_strategy": "SIMULTANEOUS",
  "ring_time": 25,
  "skip_busy": true,
  "cid_passthrough": true,
  "confirm_calls": false,
  "failover_dest_type": "VOICEMAIL",
  "failover_dest_id": "aaaabbbb-cccc-dddd-eeee-ffffffffffff",
  "member_extension_ids": [
    "11112222-3333-4444-5555-666677778888",
    "22223333-4444-5555-6666-777788889999",
    "33334444-5555-6666-7777-888899990000"
  ]
}

Field Reference

RingGroupCreate Fields

Field Type Required Default Description
group_number string Yes — The dial number for this group (1–20 characters). Example: "600". Users dial this to reach the group.
name string Yes — Descriptive name for the ring group (1–255 characters)
ring_strategy enum No SIMULTANEOUS How members are rung: SIMULTANEOUS, SEQUENTIAL, or ROUND_ROBIN
ring_time int No 25 Total ring time in seconds before failover (5–300). For simultaneous, this is how long all phones ring. For sequential/round-robin, this is the maximum total time across all members.
ring_time_per_member int No 15 Seconds to ring each member in sequential and round-robin strategies (5–120). Ignored for simultaneous.
skip_busy boolean No true Skip members who are currently on a call. When false, busy members' phones will receive a call-waiting signal.
cid_passthrough boolean No true When true, the original external caller ID is displayed on ringing phones. When false, the ring group name and number are displayed instead.
confirm_calls boolean No false Require the answering party to press 1 to accept the call. Prevents voicemail systems from "answering" the ring group call. See Call Confirmation.
failover_dest_type string No null Destination type if no member answers (max 20 characters). Accepts the same destination types as inbound routes: EXTENSION, VOICEMAIL, QUEUE, IVR, etc.
failover_dest_id UUID No null UUID of the failover destination object
moh_prompt_id UUID No null Audio prompt played as music on hold while the caller waits (used during sequential/round-robin between members)
member_extension_ids list[UUID] No [] Ordered list of extension UUIDs. Order determines ring sequence for sequential and round-robin strategies.

Response Fields

The API returns all create fields plus the following system-managed fields.

Field Type Description
id UUID Unique identifier for the ring group
tenant_id UUID The tenant that owns this ring group
is_active boolean Whether the ring group is active and has at least one member
created_at datetime Timestamp when the ring group was created
updated_at datetime Timestamp of the last modification

Members in the response include both extension_id and position fields, indicating each member's order in the group.

Member Management

Members are the extensions that will ring when the group is called. The order of the member_extension_ids list determines the ring sequence for sequential and round-robin strategies. For simultaneous ringing, order does not matter.

Managing Members

  • Add members — Include new extension IDs in the member_extension_ids list when updating the group.
  • Remove members — Update the group with a member_extension_ids list that omits the extensions you want to remove.
  • Reorder members — Change the order of IDs in the list. The first ID is position 1, the second is position 2, and so on.
Extension Availability
An extension can be a member of multiple ring groups. If the extension is already on a call when a ring group fires, the skip_busy setting determines whether it is skipped or receives a call-waiting notification.

Failover Destinations

When the ring time expires and no member has answered, the call is sent to the failover destination. If no failover is configured, the caller hears a busy signal and the call is dropped.

Common Failover Configurations

Failover Destination Use Case
Voicemail Caller leaves a message for the team. Most common failover.
Queue Escalate to a formal queue with hold music and agents.
Another Ring Group Cascade to a backup team if the primary team is unavailable.
IVR Return the caller to a menu to choose an alternative option.
Extension Send to a specific person (e.g., office manager) as a last resort.

Call Confirmation

When confirm_calls is enabled, the person who picks up hears a brief prompt: "Press 1 to accept this call." If they press 1, the call is connected. If they do not press a key (or press any other key), the system treats it as a non-answer and moves to the next member or failover.

This feature solves two problems:

  • Voicemail interception — When a ring group includes mobile phones or extensions with voicemail, the voicemail system may "answer" the call before a human does. Call confirmation ensures only a real person accepts the call.
  • Selective answering — Team members who are busy but not marked as "on a call" (e.g., in a meeting) can decline by simply not pressing 1.
Caller Experience
When call confirmation is enabled, the caller hears continued ringing (or music on hold if configured) while the answering party decides. This adds a few seconds of delay. Only enable call confirmation when voicemail interception is a real problem.

Caller ID Passthrough vs Override

The cid_passthrough setting controls what appears on the ringing phone's display.

Setting Phone Display When to Use
true (default) Original caller's name and number (e.g., "Jane Smith 5551234567") When team members need to see who is calling before answering
false Ring group name and number (e.g., "Front Desk 600") When team members need to know which group the call came through rather than who is calling
Combine with CID Prefix
For the best of both worlds, leave cid_passthrough enabled and use a CID name prefix on the inbound route instead. This shows both the caller's identity and which line rang (e.g., "Sales: Jane Smith").

Music on Hold

The moh_prompt_id field specifies an audio prompt to play while the caller waits during sequential or round-robin ringing. Between member attempts, the caller hears this audio instead of silence. For simultaneous ringing, the caller hears standard ringback tone on all phones and MOH is not used.

If no MOH prompt is specified, the system default hold music is used. See Audio Prompts to upload custom audio files.

Common Use Cases

Sales Team

Three sales reps share a number. Calls ring all phones simultaneously. If nobody answers in 20 seconds, the call goes to a shared sales voicemail box. 

{
  "group_number": "610",
  "name": "Sales Team",
  "ring_strategy": "SIMULTANEOUS",
  "ring_time": 20,
  "skip_busy": true,
  "cid_passthrough": true,
  "failover_dest_type": "VOICEMAIL",
  "failover_dest_id": "...",
  "member_extension_ids": ["ext-1-uuid", "ext-2-uuid", "ext-3-uuid"]
}

Support Team (Fair Distribution)

Five support staff need calls distributed evenly. Round-robin ensures each person gets roughly the same number of calls. If nobody answers, escalate to the support queue. 

{
  "group_number": "620",
  "name": "Support Team",
  "ring_strategy": "ROUND_ROBIN",
  "ring_time": 60,
  "ring_time_per_member": 15,
  "skip_busy": true,
  "cid_passthrough": true,
  "confirm_calls": false,
  "failover_dest_type": "QUEUE",
  "failover_dest_id": "...",
  "member_extension_ids": ["ext-a", "ext-b", "ext-c", "ext-d", "ext-e"]
}

After-Hours Escalation

An after-hours ring group calls the on-call manager first, then the backup manager. Sequential strategy ensures the primary contact is tried first. Call confirmation prevents the mobile voicemail from intercepting the call. 

{
  "group_number": "699",
  "name": "After Hours On-Call",
  "ring_strategy": "SEQUENTIAL",
  "ring_time": 90,
  "ring_time_per_member": 30,
  "skip_busy": false,
  "cid_passthrough": true,
  "confirm_calls": true,
  "failover_dest_type": "VOICEMAIL",
  "failover_dest_id": "...",
  "member_extension_ids": ["manager-uuid", "backup-uuid"]
}
