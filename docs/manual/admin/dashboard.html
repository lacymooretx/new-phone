Dashboard Guide

Real-time system overview showing call activity, extension status, queue health, and system metrics at a glance.

Overview

The Dashboard is the first page administrators see after logging in. It provides a real-time snapshot of the entire phone system, consolidating key metrics into a single view. From active call counts to system health indicators, the dashboard gives you immediate situational awareness of your telephony environment.

Tenant-Scoped View
The dashboard displays data scoped to the current tenant. MSP administrators can switch between tenants using the tenant selector in the top navigation bar to view metrics for each customer individually. A global MSP overview dashboard is planned for a future release.

Dashboard Layout

The dashboard is organized into widget cards arranged in a responsive grid. Each card focuses on a specific aspect of the system and updates in real time via WebSocket connections.

Widget Sections

Section Position Description
Call Activity Top row, left Active calls, total calls today, average duration
Extension Status Top row, right Registration counts by state
Queue Statistics Middle row, left Queue health and agent availability
System Health Middle row, right Subsystem status indicators
Recent Calls Bottom row, left Latest call activity feed
Voicemail Summary Bottom row, right Unread and total voicemail counts

Call Activity Metrics

The Call Activity widget shows real-time and daily call statistics. These metrics are derived from FreeSWITCH channel data and CDR records.

Metrics Displayed

Metric Source Description
Active Calls FreeSWITCH (real-time) Number of calls currently in progress across all extensions, queues, and trunks
Calls Today CDR database Total inbound and outbound calls since midnight (tenant local time)
Avg Duration CDR database Average call duration for today's completed calls, displayed in minutes and seconds
Missed Calls CDR database Calls that rang but were not answered, went to voicemail, or were abandoned in queue

The active calls count updates every 2 seconds via WebSocket push. Daily aggregates (calls today, average duration, missed calls) refresh every 30 seconds.

Extension Status Overview

The Extension Status widget provides a breakdown of all extensions in the tenant, grouped by their current registration and availability state.

Extension States

State Indicator Meaning
Registered Green Phone is online and available to receive calls
On a Call Blue Phone is registered and currently on an active call
Ringing Amber Phone is currently ringing with an incoming call
DND Red Phone is registered but Do Not Disturb is enabled
Offline Gray Phone is not registered (powered off, network issue, or unprovisioned)

The widget shows a count for each state and a visual bar chart representing the proportion. Click any state label to filter the Extensions page to show only extensions in that state.

Queue Statistics

The Queue Statistics widget gives a high-level view of call queue health across all queues in the tenant. Detailed per-queue metrics are available on the Queues page.

Metrics Displayed

Metric Description
Callers Waiting Total number of callers currently waiting in all queues combined
Available Agents Number of queue agents logged in and available to take calls
Longest Wait Duration of the caller who has been waiting the longest, across all queues
Avg Wait Time Average time callers spent in queue before being answered (today)
Abandoned Number of callers who hung up while waiting in queue (today)
SLA % Percentage of calls answered within the configured SLA threshold (e.g., 80% in 20 seconds)
Queue Alert
When the longest wait time exceeds the configured SLA threshold, the Callers Waiting metric changes to a red alert state. This visual cue helps administrators quickly identify when queue service levels are degraded.

System Health Indicators

The System Health widget displays the status of each core platform subsystem. It queries the /api/v1/health endpoint and presents the results as colored status indicators.

Status Indicators

Subsystem Healthy Degraded Down
API Responding normally High latency (> 1s P95) Not responding
FreeSWITCH ESL connection ok High channel count (> 80% capacity) ESL connection refused
PostgreSQL Queries responding High connection pool usage Connection refused
Redis PING succeeds High memory usage Connection refused

Each subsystem shows a green, amber, or red dot. Hovering over a subsystem reveals additional details such as response time and last check timestamp. The health check runs every 15 seconds.

Recent Call Activity

The Recent Calls widget shows a live feed of the last 20 calls. Each entry displays the caller, callee, direction (inbound/outbound/internal), duration, and outcome (answered, missed, voicemail).

  • Inbound calls show the external caller number and the destination extension or queue
  • Outbound calls show the originating extension and the dialed number
  • Internal calls show both extension numbers

Click any call entry to navigate to the full CDR detail view, which includes call flow steps, recording playback (if available), and disposition codes.

Voicemail Summary

The Voicemail Summary widget shows aggregate voicemail statistics for the tenant.

Metric Description
New Messages Total unread voicemails across all mailboxes in the tenant
Total Messages All voicemails (read + unread) across all mailboxes
Storage Used Approximate storage consumption for voicemail audio files
Mailboxes with New Number of individual mailboxes that have at least one unread message

The widget includes a link to the full Voicemail management page for detailed message browsing and playback.

Customizing the Dashboard Coming Soon

A future release will allow administrators to customize the dashboard layout, including the ability to rearrange widgets, choose which metrics are displayed, add custom widgets, and save per-user dashboard preferences.

  • Drag-and-drop widget rearrangement
  • Widget visibility toggles
  • Custom date range selectors for aggregate metrics
  • Saved dashboard presets per user role
  • Full-screen mode for wall-mounted displays

Auto-Refresh Behavior

The dashboard uses a combination of WebSocket push and polling to keep data current. Different widgets refresh at different intervals based on how time-sensitive their data is.

Widget Update Method Interval
Active Calls WebSocket push ~2 seconds
Extension Status WebSocket push ~2 seconds (on state change)
Queue Statistics WebSocket push ~5 seconds
System Health Polling 15 seconds
Recent Calls WebSocket push On each new call event
Voicemail Summary Polling 60 seconds
Calls Today / Avg Duration Polling 30 seconds
WebSocket Connection
If the WebSocket connection drops (network interruption, server restart), the dashboard automatically falls back to polling and attempts to reconnect. A yellow "Reconnecting..." indicator appears in the top right corner of the dashboard during reconnection attempts.
