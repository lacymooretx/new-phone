UI Guide

Tips and patterns for navigating the web application, using dark mode, keyboard shortcuts, and common UI interactions.

Overview

The Aspendora Connect web application is built with React and TypeScript, providing a modern, responsive interface for managing your phone system. This guide covers the common UI patterns, navigation structure, keyboard shortcuts, and tips to help you work efficiently.

Consistent Patterns
The application uses consistent UI patterns across all pages. Once you learn how to create, edit, and delete items on one page (e.g., Extensions), the same patterns apply to every other resource type (Ring Groups, Queues, IVR Menus, etc.).

Navigation

The application uses a sidebar-based navigation layout with breadcrumbs for context.

Sidebar

The left sidebar contains all primary navigation links organized into groups: Telephony, Connectivity, Reports, System, Security, MSP, and Interface. The sidebar is collapsible on desktop by clicking the hamburger icon, and slides in as an overlay on mobile devices.

  • Click a group label to expand or collapse that section
  • The currently active page is highlighted in the sidebar
  • The sidebar remembers which groups are expanded across page navigation
  • On mobile, the sidebar opens from the left edge and can be dismissed by tapping the overlay

Breadcrumbs

Breadcrumbs appear at the top of every page, showing the current location in the navigation hierarchy. Each breadcrumb segment is a clickable link. The format is: 

Admin Manual / Section / Current Page

On detail pages (e.g., editing a specific extension), the breadcrumb includes the list page as a parent: 

Admin Manual / Telephony / Extensions / Extension 1001

Dark Mode

The application supports both light and dark themes. The theme preference is stored in the browser and persists across sessions.

Theme Toggle

Click the moon/sun icon in the bottom of the sidebar (or in the top navigation bar on mobile) to switch between light and dark mode. The change applies immediately across all pages.

System Preference

On first visit, the application respects your operating system's theme preference (prefers-color-scheme media query). If your OS is set to dark mode, the application will default to dark mode. Once you manually toggle the theme, your explicit preference overrides the system setting.

Command Palette / Quick Search Coming Soon

A future release will add a command palette accessible via Cmd+K (macOS) or Ctrl+K (Windows/Linux). The command palette will allow you to:

  • Search across all pages and resources by name or number
  • Jump to any page in the application instantly
  • Search extensions, users, DIDs, and queues by name or number
  • Execute quick actions (create extension, view dashboard, etc.)
  • Access recently visited pages

Keyboard Shortcuts

The following keyboard shortcuts are available throughout the application to speed up common operations.

Shortcut Action Context
Cmd/Ctrl + K Open command palette Global (Coming Soon)
Escape Close modal or dialog When a modal is open
Cmd/Ctrl + S Save current form When editing a resource
N Create new resource On list pages (when no input is focused)
/ Focus search/filter input On list pages (when no input is focused)
J / K Move down / up in list On list pages (when no input is focused)
Enter Open selected item When an item is highlighted in a list
? Show keyboard shortcut help Global (when no input is focused)
Shortcut Safety
Keyboard shortcuts that trigger destructive actions (delete, bulk operations) always require confirmation via a dialog. Single-key shortcuts (N, /, J, K) are disabled when a text input, textarea, or select element is focused.

Table Interactions

Resource list pages (Extensions, Users, CDRs, etc.) use data tables with consistent interactive features.

Sorting

Click any column header to sort the table by that column. Click again to toggle between ascending and descending order. An arrow icon indicates the current sort direction. The default sort varies by page (e.g., Extensions sort by extension number, CDRs sort by date descending).

Filtering

Most list pages include a search/filter bar at the top. Type to filter results in real time. Some pages offer advanced filter dropdowns for specific fields (e.g., filter CDRs by direction, filter extensions by status). Active filters are shown as removable chips below the search bar.

Pagination

Large result sets are paginated with controls at the bottom of the table. You can adjust the number of rows per page (10, 25, 50, 100) and navigate between pages. The current page and total count are always displayed.

Row Selection

Tables that support bulk operations have checkboxes in the first column. Select individual rows by clicking their checkbox, or use the header checkbox to select all visible rows. Bulk actions (delete, export) appear in a toolbar above the table when rows are selected.

Form Patterns

All create and edit forms follow consistent patterns across the application.

Creating a Resource

  1. Navigate to the resource list page (e.g., Extensions)
  2. Click the Create button (or press N)
  3. Fill in the required fields (marked with an asterisk)
  4. Click Save (or press Cmd/Ctrl + S)
  5. A success notification appears and you are redirected to the list or detail page

Editing a Resource

  1. Click the resource name in the list to open the detail view
  2. Click Edit to enter edit mode
  3. Modify the desired fields
  4. Click Save to apply changes, or Cancel to discard
  5. A success notification confirms the update

Deleting a Resource

  1. Click the delete icon (trash) on the list page, or the Delete button on the detail page
  2. A confirmation dialog appears showing what will be deleted
  3. Type the resource name or click Confirm to proceed
  4. The resource is deleted and you are redirected to the list page
Deletion is Permanent
Deleting a resource is irreversible. The confirmation dialog shows any dependent resources that will also be affected (e.g., deleting an extension removes its voicemail box, queue memberships, and ring group memberships).

Responsive Design

The application adapts to different screen sizes with three primary breakpoints.

Breakpoint Width Layout Changes
Desktop ≥ 1024px Full sidebar visible, three-column layout (sidebar + content + page TOC)
Tablet 768px – 1023px Sidebar collapses to icons, page TOC hidden, two-column layout
Mobile < 768px Sidebar becomes slide-out overlay, single-column layout, tables scroll horizontally

On smaller screens, data tables become horizontally scrollable to prevent content truncation. The softphone panel resizes to fit mobile viewports. All interactive elements maintain minimum touch target sizes of 44x44 pixels.

Notifications and Toasts

The application uses toast notifications to provide feedback on user actions. Toasts appear in the top-right corner and automatically dismiss after a few seconds.

Type Color Duration Usage
Success Green 3 seconds Resource created, updated, or deleted successfully
Error Red Persistent (manual dismiss) API errors, validation failures, network issues
Warning Amber 5 seconds Partial success, non-critical issues, deprecation notices
Info Blue 4 seconds Informational messages, status updates, tips

Error toasts remain visible until manually dismissed to ensure you do not miss important error messages. All other toast types can be dismissed early by clicking the close button.

Loading States

The application provides visual feedback during data loading to keep you informed of what is happening.

Loading Indicators

  • Page skeleton — On initial page load, a skeleton placeholder mimics the page layout until data arrives. This prevents layout shifts and gives immediate visual structure.
  • Spinner — Small inline spinners appear next to buttons after clicking to indicate the action is in progress (e.g., saving a form).
  • Progress bar — A thin progress bar at the top of the page indicates page-level loading or navigation transitions.
  • Table loading — When table data is refreshing (sort change, filter, pagination), a subtle overlay dims the table content to indicate loading while preserving the current view.
  • Disabled buttons — Submit and action buttons are disabled while their operation is in progress to prevent duplicate submissions.
Optimistic Updates
Some operations use optimistic updates, where the UI updates immediately before the API confirms the change. If the API call fails, the UI reverts to the previous state and an error toast is shown. This approach provides a snappier user experience for common operations.
