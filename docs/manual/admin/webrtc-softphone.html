WebRTC Softphone

Browser-based SIP calling via WebSocket Secure — make and receive calls directly from the web UI without a desk phone.

Overview

The WebRTC Softphone is a built-in browser-based phone that connects to FreeSWITCH via WebSocket Secure (WSS). It allows users to make and receive calls directly from the web application without any additional software or hardware. The softphone registers as a SIP endpoint using the user's assigned extension, and all signaling travels over encrypted WebSocket connections while media is encrypted with SRTP.

Encryption by Default
All WebRTC connections use mandatory encryption. Signaling flows over WSS (WebSocket over TLS) and media is protected by SRTP with DTLS key exchange. There is no option to use unencrypted connections, consistent with the platform's security-first design.

System Requirements

The WebRTC softphone runs entirely in the browser. The following requirements must be met for proper operation.

Requirement Details
Browser Chrome 90+, Firefox 88+, Edge 90+, Safari 15+
Protocol Page must be served over HTTPS (required for microphone access)
Microphone Browser must have permission to access the microphone
Network Outbound WSS (port 443 or 7443) and UDP/TCP for STUN/TURN
Audio Output Speakers or headset for call audio playback
Safari Limitations
Safari on iOS does not support background WebRTC audio. Incoming call notifications will not ring if the browser tab is in the background. For mobile use, the native Flutter app (coming soon) is recommended.

Getting Started

To use the WebRTC softphone, navigate to the Softphone section in the web application. The softphone panel can be accessed from the phone icon in the top navigation bar or from the sidebar under Interface → WebRTC Softphone.

Automatic Registration

When the softphone panel opens, it automatically fetches WebRTC credentials from the API and registers with FreeSWITCH via WSS. The registration process takes 1-3 seconds. Once registered, the status indicator turns green and displays "Registered."

  1. Open the Softphone panel (click the phone icon or navigate via sidebar)
  2. Grant microphone permission when prompted by the browser
  3. Wait for the status to show Registered (green indicator)
  4. You are ready to make and receive calls

Making Calls

There are two ways to initiate outbound calls from the softphone.

Using the Dial Pad

Click the dial pad button to open the numeric keypad. Enter the number you want to call (extension number for internal calls, or full phone number for external calls) and press the green call button. You can also type the number directly using your keyboard.

Click-to-Call from Contacts

Throughout the web application, phone numbers and extension numbers are clickable. Clicking a phone number in the contacts list, call history, or any other page automatically opens the softphone and initiates the call. This works for both internal extensions and external numbers.

Receiving Calls

When your extension receives an incoming call, the softphone displays a notification with the caller's information.

Incoming Call Notification

  • Caller name — Display name from caller ID or contact match
  • Caller number — The calling party's phone number or extension
  • Ring type — Whether the call is direct, from a queue, or from a ring group

Answer or Reject

Click the green Answer button to accept the call, or the red Reject button to send the call to voicemail (or the next destination in the call flow). If the call is not answered within the configured ring timeout, it follows the extension's no-answer destination.

Call Controls

During an active call, the softphone displays control buttons for common call operations.

Control Action Description
Hold Toggle Places the call on hold with music. Press again to resume.
Mute Toggle Mutes your microphone. The other party cannot hear you. Press again to unmute.
Transfer Opens transfer dialog Blind transfer (enter number, press send) or attended transfer (call first, then complete).
DTMF Opens keypad Send DTMF tones during a call (for IVR navigation, conference PINs, etc.).
Hang Up Ends call Terminates the active call.

Audio Device Selection

The softphone allows you to select which audio input (microphone) and output (speaker) device to use. This is useful when you have multiple audio devices connected, such as a USB headset and built-in laptop speakers.

Changing Devices

Click the gear icon in the softphone panel to open audio settings. You will see dropdown menus for:

  • Microphone — Select the input device for your voice
  • Speaker — Select the output device for call audio
  • Ringer — Select the output device for incoming ring tones (Chrome only)

Device changes take effect immediately, even during an active call. Your selection is saved in browser local storage and persists across sessions.

Registration Status

The softphone displays its SIP registration status in the top bar of the softphone panel.

Status Indicator Meaning
Registered Green Successfully registered with FreeSWITCH; ready to make and receive calls
Registering Amber Attempting to register; waiting for server response
Unregistered Gray Not connected; the softphone is idle or the panel was just opened
Error Red Registration failed; check credentials, network, or FreeSWITCH status

Call Timer

During an active call, a timer displays in the softphone panel showing the elapsed call duration in MM:SS format (or HH:MM:SS for calls exceeding one hour). The timer starts when the call is answered and stops when either party hangs up.

WebRTC Credentials

The softphone obtains its SIP credentials automatically from the API when the panel opens. These credentials are temporary and scoped to the user's session. The API endpoint returns a WebRTCCredentials object.

WebRTCCredentials Schema

Field Type Description
sip_username str SIP authentication username for WebSocket registration
sip_password str Temporary SIP password (rotated per session)
sip_domain str FreeSWITCH domain for the SIP REGISTER request
wss_url str WebSocket Secure endpoint URL (e.g., wss://pbx.example.com:7443)
extension_number str The extension number assigned to this user (e.g., "1001")
extension_id UUID The unique identifier of the extension in the database
display_name str Caller ID name sent in outbound SIP INVITE headers
Credential Security
The sip_password is temporary and rotated each time the softphone panel is opened. It is never stored persistently on the client. The credential endpoint requires a valid JWT access token, ensuring only authenticated users can obtain SIP credentials.

Troubleshooting

Common issues and their solutions when using the WebRTC softphone.

Issue Likely Cause Resolution
No audio (both directions) Microphone permission denied Click the lock/site-settings icon in the browser address bar and allow microphone access. Reload the page.
Can hear but cannot be heard Wrong microphone selected or muted Check audio device settings in the softphone. Verify the correct microphone is selected and not muted at the OS level.
One-way audio (caller hears you, you hear nothing) Speaker device misconfigured or NAT issue Check speaker device selection. If correct, this may be a NAT/firewall issue blocking inbound RTP. Verify TURN server configuration.
Registration failure (Error status) Invalid credentials or network block Refresh the page to obtain new credentials. Check that WSS port (7443) is not blocked by corporate firewall or proxy.
Echo during calls Speaker audio feeding back into microphone Use a headset instead of speakers. If using speakers, reduce volume and increase distance from microphone.
Choppy or robotic audio Network congestion or high latency Check network connection quality. Ensure sufficient bandwidth (100 Kbps per call minimum). Close bandwidth-heavy applications.
Call drops after a few seconds SRTP negotiation failure Check browser console for ICE/DTLS errors. Verify FreeSWITCH SRTP configuration matches the WebRTC requirements.
No incoming call notification Tab in background (Safari) or notifications blocked Keep the tab in the foreground, or allow browser notifications for the site. On Safari/iOS, use the native app instead.

Browser Compatibility

The WebRTC softphone has been tested on the following browsers. Other Chromium-based browsers (Brave, Vivaldi, Opera) generally work but are not officially supported.

Browser Minimum Version Audio Output Selection Background Ringing Status
Google Chrome 90+ Yes Yes Fully Supported
Microsoft Edge 90+ Yes Yes Fully Supported
Mozilla Firefox 88+ No (uses system default) Yes Supported
Safari (macOS) 15+ No Limited Supported
Safari (iOS) 15+ No No Limited
Recommended Browser
Google Chrome provides the best WebRTC experience with full audio device selection, background ringing support, and the most reliable SRTP/DTLS implementation. We recommend Chrome for primary softphone use.
