Coming Soon

Planned features and capabilities on the Aspendora Connect roadmap, organized by category.

Overview

The Aspendora Connect is under active development with a comprehensive feature roadmap. This page lists all planned features that are not yet implemented, grouped by category. Features marked with Coming Soon are on the roadmap and will be added in future releases. Priorities and timelines may shift based on customer feedback and operational needs.

Feature Requests
If you have specific feature requests or want to influence roadmap priorities, contact your MSP administrator. Feature requests are tracked and prioritized based on demand across the tenant base.

Mobile & Desktop

Flutter Mobile App Coming Soon

A native mobile application built with Flutter for iOS and Android. On iOS, the app will integrate with CallKit to provide native call UI, lock-screen call notifications, and Siri integration. On Android, it will use ConnectionService for native dialer integration and incoming call handling even when the app is in the background.

Electron Desktop App Coming Soon

A cross-platform desktop application for Windows, macOS, and Linux. The desktop app will run as a system tray application with global keyboard shortcuts for answering calls, click-to-call from any application, and desktop notifications that work even when the browser is closed.

Phone Provisioning

Auto-Provision Desk Phones Coming Soon

Zero-touch provisioning for Yealink, Polycom, and Cisco desk phones. Phones will automatically discover the provisioning server via DHCP Option 66 or DNS SRV records, download their configuration, and register with FreeSWITCH. Administrators will be able to assign phones to extensions, configure button layouts, and push firmware updates from the web UI.

DPMA Replacement for Sangoma P-Series Coming Soon

A Rust-based replacement for Digium Phone Module for Asterisk (DPMA) that provides native support for Sangoma P-series phones. This service will handle phone discovery, provisioning, presence (BLF), and feature key synchronization without depending on Asterisk or the proprietary DPMA module.

Desk Phone XML Apps Coming Soon

On-phone applications rendered through the desk phone's XML browser for Yealink, Polycom, and Cisco models. These apps will include a corporate directory with search, visual voicemail with message playback controls, a parking lot panel showing occupied and available slots, and a queue dashboard for agents to view queue status and log in/out directly from their phone screen.

Integrations

ConnectWise PSA Coming Soon

Deep integration with ConnectWise PSA for managed service providers. The platform will automatically create tickets from missed calls, voicemails, and completed calls. It will map extensions to ConnectWise companies, attach call recordings to tickets, and log call activity as time entries. Inbound caller ID will trigger screen pops showing the matching ConnectWise company and open tickets.

Zendesk Integration Coming Soon

Connect the phone system with Zendesk for customer support workflows. Inbound calls will trigger ticket lookup and screen pops, call recordings will be attached to tickets automatically, and agents can click-to-call directly from Zendesk ticket views.

Salesforce Integration Coming Soon

CRM integration with Salesforce to log call activity, match callers to contacts and accounts, and provide sales teams with click-to-call from within Salesforce. Call recordings and transcriptions will be linked to the appropriate Salesforce records.

Microsoft Teams Integration Coming Soon

Presence synchronization between the phone platform and Microsoft Teams. When a user is on a Teams call, their phone extension will show busy. Direct routing will allow Teams users to make and receive PSTN calls through the platform's SIP trunks.

Slack Integration Coming Soon

Slack notifications for missed calls, voicemails, and queue alerts. Users will receive DMs when they miss a call, and channels can be configured to receive queue overflow alerts. A Slack slash command will allow click-to-call directly from Slack.

Advanced Call Center

Dashboards & Analytics Coming Soon

Real-time and historical call center dashboards with wallboard mode for display screens. Metrics will include service level adherence, average handle time, abandonment rates, agent utilization, and hourly call volume trends with customizable date ranges and drill-down capabilities.

Workforce Management Coming Soon

Schedule management for call center agents, including shift planning, break scheduling, and adherence monitoring. The system will forecast call volumes based on historical data and recommend staffing levels to meet SLA targets.

QA Scorecards Coming Soon

Quality assurance scoring for recorded calls. Supervisors will create customizable scorecards with weighted criteria, score agent calls, and track quality trends over time. Scores can be linked to specific call recordings for agent coaching sessions.

Disposition Codes Coming Soon

Configurable disposition codes that agents select after each call to categorize the outcome (e.g., sale, support resolved, callback needed, wrong number). Disposition data feeds into reporting dashboards and can trigger automated follow-up workflows.

Scheduled Callbacks Coming Soon

Allow callers waiting in queue to request a callback instead of holding. The system will maintain the caller's position in the virtual queue and automatically dial them back when an agent becomes available, preserving their original wait position.

Post-Call Surveys Coming Soon

Automated IVR surveys presented to callers after a call ends. Callers rate their experience using DTMF input or voice, and results are tracked per agent, per queue, and over time to measure customer satisfaction trends.

Wrap-Up Codes Coming Soon

Configurable wrap-up time after each queue call during which the agent is not offered new calls. Agents use this time to complete notes, set disposition codes, and finish post-call tasks. Wrap-up duration is configurable per queue.

AI Features

Transcription Coming Soon

Automatic speech-to-text transcription of call recordings. Transcriptions will be searchable, allowing administrators to find calls by spoken content. Multiple STT providers will be supported including OpenAI Whisper, Deepgram, and Google Speech.

Summarization Coming Soon

AI-generated summaries of call recordings that extract key discussion points, action items, and decisions. Summaries will be attached to CDR records and can be included in ConnectWise ticket notes or email notifications.

Sentiment Analysis Coming Soon

Real-time and post-call sentiment scoring that detects caller emotion throughout the conversation. Supervisors can monitor live calls flagged with negative sentiment and intervene if needed. Historical sentiment data feeds into quality assurance reporting.

AI Voice Agent System Coming Soon

A full conversational AI agent modeled after a multi-provider STT/LLM/TTS pipeline. The voice agent handles inbound calls with natural conversation, tool calling for CRM lookups and ticket creation, barge-in support for interrupting the AI mid-sentence, and seamless handoff to live agents when needed. Multiple AI providers will be supported including OpenAI, Deepgram, Google Gemini, and ElevenLabs.

AI Compliance Monitoring Coming Soon

Automated monitoring of call recordings for compliance with regulatory requirements and company policies. The system will flag calls where required disclosures were missed, detect potential PCI violations (credit card numbers spoken aloud), and verify script adherence for regulated industries.

SMS & Messaging

WhatsApp Business Coming Soon

Integration with the WhatsApp Business API for omnichannel customer communication. Incoming WhatsApp messages will route to queues alongside voice calls and SMS, with conversation threading and shared inbox support for team collaboration.

Facebook Messenger Coming Soon

Connect Facebook Messenger to the queue system for businesses that receive customer inquiries through Facebook. Messages will be handled by the same agents and queues as voice and SMS, providing a unified communication experience.

Email-to-Queue Coming Soon

Route inbound emails to call center queues for agents to handle alongside voice and messaging channels. Emails will be assigned to agents using the same routing strategies as calls (round-robin, least-recent, skills-based), with response templates and SLA tracking.

Security & Compliance

STIR/SHAKEN Coming Soon

Implementation of the STIR/SHAKEN caller ID authentication framework. Outbound calls will be signed with attestation certificates, and inbound calls will display verification status (full, partial, or gateway attestation) to help users identify legitimate callers and filter robocalls.

Robocall Filtering Coming Soon

Intelligent call screening that blocks known robocall numbers, challenges suspected spam callers with CAPTCHA-style audio prompts, and maintains per-tenant block/allow lists. The system will use reputation databases and STIR/SHAKEN attestation data to score incoming calls.

TCPA Compliance Coming Soon

Tools to help organizations comply with the Telephone Consumer Protection Act, including do-not-call list management, consent tracking for outbound campaigns, and automated opt-out processing. The system will enforce calling hour restrictions and maintain auditable records of consent.

10DLC Toolkit Coming Soon

A complete 10-digit long code compliance toolkit for SMS/MMS messaging. This includes brand registration, campaign registration with The Campaign Registry (TCR), opt-in form generation, privacy policy and terms of service templates, and throughput monitoring to stay within approved message volumes.

Infrastructure

High Availability Coming Soon

Active/passive and active/active clustering for all platform components. FreeSWITCH will support failover pairs with shared registration state, PostgreSQL will use streaming replication with automatic failover, and the API layer will run behind a load balancer with health-check-based routing.

Disaster Recovery Coming Soon

Automated disaster recovery with configurable RPO (recovery point objective) and RTO (recovery time objective) targets. The system will support cross-region replication of databases, media storage, and configuration, with automated failover procedures and regular DR testing capabilities.

Multi-Region Deployment Coming Soon

Deploy platform instances across multiple geographic regions to reduce latency for distributed workforces and meet data residency requirements. Each region will operate independently with cross-region synchronization for global directory and configuration.

Status Page Coming Soon

A public-facing status page showing real-time health of all platform components. Tenants and their end users can subscribe to incident notifications via email or webhook. The status page will include historical uptime metrics and incident post-mortems.

Billing & Business

Usage Metering Coming Soon

Per-tenant usage tracking for billable resources including call minutes (inbound, outbound, toll-free), SMS/MMS messages, recording storage, DID rental, and API calls. Usage data will be available in real time via the admin dashboard and exportable for billing systems.

Rate Decks Coming Soon

Configurable per-destination rate tables for call pricing. MSP administrators will be able to define markup rules on top of carrier costs, set per-tenant rate overrides, and generate cost-vs-revenue reports for each tenant.

Pax8 / ConnectWise Billing Integration Coming Soon

Direct integration with Pax8 and ConnectWise billing systems to automate invoice generation for MSP customers. Usage data from the metering system will flow into billing agreements, with support for tiered pricing, bundled plans, and overage charges.

Advanced Features

White-Label Branding Coming Soon

Per-tenant branding customization including logos, color schemes, custom domain names, and branded email templates. MSPs will be able to resell the platform under their own brand with no visible reference to the underlying platform.

Inter-Tenant Calling Coming Soon

Direct extension-to-extension calling between tenants on the same platform without routing through external SIP trunks. This reduces cost and latency for MSPs whose customers frequently call each other, with configurable per-tenant permissions for which tenants can be reached.

Boss/Admin (Executive/Assistant) Coming Soon

Executive/assistant call filtering where an assistant screens calls for an executive. The assistant's phone rings first, and they can announce the caller, put the call through, take a message, or route to voicemail. Multiple assistants can be assigned to one executive with priority ordering.

Camp-On / Callback on Busy Coming Soon

When a caller reaches a busy extension, they can choose to "camp on" and be automatically called back when the extension becomes available. The system monitors the target extension's state and initiates the callback within seconds of it going idle.

Multi-Site / Multi-Timezone Coming Soon

Support for tenants with offices in multiple locations and time zones. Time conditions, business hours, and holiday calendars can be configured per site. The system will route calls based on the local time at each site and support site-specific dial plans and emergency routing.

CDR Enrichment from CRM Coming Soon

Automatically enrich call detail records with data from connected CRM systems. Inbound caller phone numbers will be matched against CRM contacts to display company name, account status, and recent ticket history alongside call records, making CDR reports more actionable.

Click-to-Call Browser Extension Coming Soon

A Chrome/Firefox browser extension that detects phone numbers on any web page and adds a click-to-call overlay. Clicking a detected number initiates a call through the platform's WebRTC softphone or sends a dial command to a registered desk phone.

Developer

Developer Portal Coming Soon

A self-service developer portal with interactive API documentation (Swagger/ReDoc), API key management, usage analytics, code samples in multiple languages, and a sandbox environment for testing integrations without affecting production data.

Webhooks Coming Soon

Configurable webhook endpoints that fire on platform events such as call start/end, voicemail received, extension registration changes, and queue events. Webhooks will include HMAC signature verification, retry logic with exponential backoff, and a delivery log for debugging.

SDKs Coming Soon

Official client SDKs for Python, JavaScript/TypeScript, Go, and C# that wrap the REST API with typed models, authentication handling, pagination helpers, and retry logic. SDKs will be published to PyPI, npm, and NuGet respectively.

Plugin / Marketplace Architecture Coming Soon

An extensibility framework allowing third-party developers to build plugins that integrate with the platform. Plugins will be able to add custom call flow actions, UI panels, reporting widgets, and integration connectors. A marketplace will allow MSPs to discover and install plugins.

Other

Receptionist Console Coming Soon

A dedicated web interface designed for receptionists and operators who handle high call volumes. The console provides a streamlined view with drag-and-drop call transfer, extension presence panel, speed-dial contacts, and a call parking lot visualizer, all optimized for rapid call handling.

Hospitality Module Coming Soon

Features tailored for hotels and hospitality environments including room extension management, wake-up call scheduling, guest check-in/check-out integration, minibar and housekeeping status codes, and restricted dialing profiles for guest room phones.

Headset Integration Coming Soon

Native integration with professional headsets from Jabra, Poly, and EPOS/Sennheiser. Physical headset buttons (answer, hang up, mute, volume) will control the WebRTC softphone directly via the WebHID API, providing a seamless hardware-software experience.

Emergency / Physical Security Integration Coming Soon

Integration with physical security systems to trigger actions on emergency calls. When a 911 call is placed, the system can activate building security cameras, unlock doors for emergency responders, send notifications to designated security personnel, and log the calling location for E911 dispatch.

Recording Storage Tiering Coming Soon

Automated lifecycle management for call recordings, moving files between storage tiers based on age and access patterns. Recent recordings stay on fast storage (SSD-backed S3), older recordings move to cost-effective cold storage, and expired recordings are automatically purged based on retention policies.

Localization / i18n Coming Soon

Full internationalization of the web UI and system prompts. The platform will support multiple languages for the admin interface, end-user portal, IVR prompts, and email templates. Per-tenant language preferences will control which language is used for system-generated content.

Migration Tools Coming Soon

Automated migration tools for importing data from existing PBX systems including FreePBX, 3CX, and Asterisk. The tools will import extensions, ring groups, IVR menus, time conditions, DIDs, voicemail messages, and call recordings, with a pre-migration validation report.

Network QoS Tools Coming Soon

Built-in network quality diagnostics including jitter, latency, and packet loss measurement between endpoints and the platform. Administrators can run on-demand network assessments before deploying phones at a new site and set up continuous monitoring with alerts when quality degrades.

SIP Debugging Tools Coming Soon

A web-based SIP trace viewer for troubleshooting call setup failures, registration issues, and codec negotiation problems. The tool will capture SIP message flows in real time, display them as ladder diagrams, and allow filtering by extension, trunk, or call ID without requiring SSH access to the FreeSWITCH server.
