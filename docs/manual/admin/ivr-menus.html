IVR / Auto Attendant

Build interactive voice response menus that route callers through DTMF digit selections, with support for multi-level nesting, prompt management, and timeout handling.

Overview

An IVR (Interactive Voice Response) menu — also called an auto attendant — greets callers with a recorded prompt and waits for them to press a key on their phone. Each key maps to an action: transfer to an extension, enter a queue, jump to another IVR submenu, reach a voicemail box, or any other destination in the system.

IVR menus are the primary tool for building self-service call routing trees. A single top-level IVR can branch into multiple submenus, creating a multi-level navigation structure that handles complex organizations without human intervention.

Common Auto Attendant Pattern
Most businesses use a single top-level IVR as the main company greeting, connected to an inbound route. Callers hear "Press 1 for Sales, 2 for Support, 3 for Billing" and are routed accordingly. Each department can have its own sub-IVR for further routing.

Creating an IVR Menu

To create an IVR menu, submit a POST request to the IVR endpoint with the menu configuration and its list of digit-to-action mappings. At minimum you need a name, but a useful IVR also requires at least one greeting prompt and one or more options. 

POST /api/v1/ivr-menus

{
  "name": "Main Auto Attendant",
  "description": "Company main greeting and top-level routing",
  "greet_long_prompt_id": "aaa-bbb-ccc-111",
  "greet_short_prompt_id": "aaa-bbb-ccc-222",
  "invalid_sound_prompt_id": "aaa-bbb-ccc-333",
  "exit_sound_prompt_id": "aaa-bbb-ccc-444",
  "timeout": 10,
  "max_failures": 3,
  "max_timeouts": 3,
  "inter_digit_timeout": 2,
  "digit_len": 1,
  "enabled": true,
  "options": [
    { "digits": "1", "action_type": "queue", "action_target_id": "queue-uuid-sales", "label": "Sales" },
    { "digits": "2", "action_type": "queue", "action_target_id": "queue-uuid-support", "label": "Support" },
    { "digits": "3", "action_type": "goto", "action_target_id": "ivr-uuid-billing", "label": "Billing Menu" },
    { "digits": "0", "action_type": "extension", "action_target_id": "ext-uuid-operator", "label": "Operator" }
  ]
}

Field Reference

IVRMenuCreate Fields

Field Type Required Default Description
name string Yes — Display name for the IVR menu (1–255 characters)
description string No null Optional description of the menu's purpose
greet_long_prompt_id UUID No null Audio prompt played when a caller first enters the IVR
greet_short_prompt_id UUID No null Shorter prompt replayed on timeout or return from a submenu
invalid_sound_prompt_id UUID No null Audio played when the caller presses an unrecognized key
exit_sound_prompt_id UUID No null Audio played when the caller exits the IVR (max failures/timeouts reached)
timeout int No 10 Seconds to wait for caller input after the prompt finishes (1–300)
max_failures int No 3 Maximum invalid key presses before the IVR exits (1–10)
max_timeouts int No 3 Maximum times the caller can time out before the IVR exits (1–10)
inter_digit_timeout int No 2 Seconds to wait between digits for multi-digit input (1–30)
digit_len int No 1 Expected number of digits per selection (1–10)
exit_destination_type string No null Where to route the caller on exit (e.g., "extension", "voicemail")
exit_destination_id UUID No null UUID of the exit destination resource
enabled bool No true Whether the IVR menu is active and can receive calls
options list No [] List of IVRMenuOptionCreate objects defining digit mappings

Response Fields

The API response includes all create fields plus the following system-managed fields:

Field Type Description
id UUID Unique identifier for this IVR menu
tenant_id UUID Owning tenant (set automatically from auth context)
tts_engine string Text-to-speech engine (reserved for future TTS integration)
tts_voice string TTS voice selection (reserved for future TTS integration)
is_active bool System-computed active state (enabled and not soft-deleted)
created_at datetime Timestamp when the menu was created
updated_at datetime Timestamp of the last update
options list Full list of IVR menu options with their IDs

Menu Options (DTMF Mappings)

Each option in the options list maps a DTMF digit sequence to an action. When a caller presses a matching digit, the IVR executes the corresponding action.

IVRMenuOptionCreate Fields

Field Type Required Default Description
digits string Yes — DTMF digit(s) that trigger this option (1–10 characters, e.g. "1", "42", "*")
action_type string Yes — Action to perform: "goto", "extension", "voicemail", "queue", "ring_group", "conference", "hangup", etc.
action_target_id UUID No null UUID of the target resource (extension, queue, IVR submenu, etc.)
action_target_value string No null Free-text value for actions that do not use a UUID target
label string No null Human-readable label for display in the admin UI
position int No 0 Sort order for display purposes (does not affect call routing)

Supported Action Types

Action Type Target Field Behavior
extension action_target_id Transfer the call to the specified extension
voicemail action_target_id Send the caller directly to the specified voicemail box
queue action_target_id Place the caller into the specified call queue
ring_group action_target_id Ring the specified ring group
goto action_target_id Jump to another IVR menu (submenu navigation)
conference action_target_id Place the caller into a conference bridge
hangup — Hang up the call (no target needed)

Prompt Configuration

IVR menus support four distinct prompt slots, each serving a different purpose in the caller experience. All prompts are optional but strongly recommended for production use. Upload audio files through the Audio Prompts module before referencing them here.

Prompt Field When Played
Long Greeting greet_long_prompt_id First time the caller enters the IVR. Typically the full greeting: "Thank you for calling Acme Corp. For Sales, press 1. For Support, press 2..."
Short Greeting greet_short_prompt_id Replayed after a timeout or when the caller returns from a submenu. A shorter version: "Press 1 for Sales, 2 for Support..."
Invalid Sound invalid_sound_prompt_id When the caller presses an unrecognized key. Typically: "That is not a valid selection. Please try again."
Exit Sound exit_sound_prompt_id When max failures or max timeouts are reached and the IVR is about to exit. Typically: "Goodbye."
Long vs Short Greeting
The long greeting plays only once when the caller first enters the IVR. If the caller times out or presses an invalid key, the short greeting replays instead. This avoids forcing repeat callers to listen to the full company introduction each time. If no short greeting is set, the long greeting replays.

Timeout and Failure Handling

The IVR tracks two separate counters: timeouts (caller does nothing) and failures (caller presses an invalid key). Each counter increments independently and resets on a successful selection. When either counter reaches its maximum, the IVR exits.

Timeout Flow

  • After the greeting plays, the system waits timeout seconds for input.
  • If no input is received, the timeout counter increments by one.
  • If max_timeouts has not been reached, the short greeting replays.
  • If max_timeouts is reached, the exit sound plays and the caller is routed to the exit destination.

Failure Flow

  • When the caller presses a digit that does not match any option, the failure counter increments.
  • The invalid sound prompt plays, then the short greeting replays.
  • If max_failures is reached, the exit sound plays and the caller exits.

Exit Destination

When the IVR exits due to max failures or max timeouts, the caller is routed to the exit destination defined by exit_destination_type and exit_destination_id. Common exit destinations include an operator extension, a general voicemail box, or the company receptionist. If no exit destination is configured, the call is disconnected.

Always Set an Exit Destination
Without an exit destination, callers who cannot navigate the IVR will be disconnected silently. Always configure an exit destination — at minimum, send them to a voicemail box or operator extension.

Multi-Digit Input

By default, digit_len is set to 1, meaning the IVR expects a single key press per selection. For IVRs that need multi-digit input (such as dialing an extension number directly), increase digit_len to match the expected length.

  • digit_len: 1 — Standard single-key menus (press 1, 2, 3, etc.)
  • digit_len: 3 — Three-digit extension dial-by-number (press 100, 101, etc.)
  • digit_len: 4 — Four-digit extension ranges

The inter_digit_timeout controls how long the system waits between consecutive key presses. If the caller pauses longer than this value between digits, the system treats the collected digits as the complete input. Set this higher (3–5 seconds) for users who may dial slowly.

Nested IVRs (Submenus)

IVR menus can be chained together by using the "goto" action type with the target set to another IVR menu's UUID. This creates a multi-level menu tree where callers navigate deeper by pressing digits.

There is no hard limit on nesting depth, but best practice is to keep it to two or three levels. Deeply nested menus frustrate callers and increase abandonment rates. 

// Option in top-level IVR that leads to a submenu
{
  "digits": "3",
  "action_type": "goto",
  "action_target_id": "ivr-uuid-billing-submenu",
  "label": "Billing Department"
}

// The billing submenu IVR might have its own options:
// Press 1 for Account Balance
// Press 2 for Payment History
// Press 3 for Speak to Billing Agent
Submenu Prompts
Each submenu is a fully independent IVR with its own greeting prompts, timeouts, and failure handling. When a caller enters a submenu, that menu's long greeting plays. Design each submenu to be self-contained with its own exit destination.

TTS Integration Coming Soon

The IVR response includes tts_engine and tts_voice fields reserved for future text-to-speech integration. When available, TTS will allow you to generate IVR prompts dynamically from text instead of uploading pre-recorded audio files.

  • Define prompt text directly in the IVR configuration
  • Select from multiple TTS engines and voice options
  • Automatic prompt regeneration when text is updated
  • Support for SSML markup for fine-grained speech control
TTS Not Yet Available
Text-to-speech integration is planned for a future release. Currently, all IVR prompts must be pre-recorded audio files uploaded through the Audio Prompts module.

IVR Tree Example

The following diagram shows a typical two-level IVR structure for a mid-size company.

Inbound Call │ └─ Main IVR (greet_long: "Thank you for calling Acme Corp...") │ ├─ [1] Sales Queue │ ├─ [2] Support Queue │ ├─ [3] Billing Sub-IVR │ │ │ ├─ [1] Account Balance → Extension 410 │ ├─ [2] Payment History → Extension 411 │ └─ [3] Billing Agent → Ring Group "Billing" │ ├─ [4] Company Directory (digit_len: 3) │ ├─ [0] Operator → Extension 100 │ └─ [timeout/max_failures] → Exit → Operator Voicemail
