Paging & Intercom

Configure page groups for one-way announcements broadcast through phone speakers, or two-way intercom for hands-free conversations.

Overview

Paging allows a user to dial a page group number and broadcast an audio announcement through the speakers of all phones in the group. Unlike a regular call, the receiving phones auto-answer on speaker without anyone picking up the handset. This is the phone-system equivalent of a PA (public address) system.

The platform supports two paging modes:

  • One-way paging (ONE_WAY) — The initiator speaks and all member phones listen. Members cannot respond. This is used for announcements: "Lunch is ready in the break room" or "Customer pickup at the front desk."
  • Two-way intercom (TWO_WAY) — The initiator speaks and the member phones auto-answer with both speaker and microphone active. Members can respond immediately, creating a hands-free conversation. This is used for quick point-to-point or group communication.
Phone Compatibility
Paging requires desk phones that support auto-answer via SIP headers. Most modern Yealink, Polycom, and Cisco phones support this feature. Softphones (WebRTC, mobile) may not auto-answer and will ring normally instead. Verify auto-answer support with your handset models before deploying paging groups.

Creating a Page Group

Create a page group by submitting a POST request with the group configuration and member list. The required fields are name and page_number. 

POST /api/v1/page-groups

{
  "name": "Warehouse PA",
  "page_number": "700",
  "description": "All warehouse floor phones",
  "page_mode": "ONE_WAY",
  "timeout": 60,
  "members": [
    { "extension_id": "ext-uuid-warehouse-1", "position": 1 },
    { "extension_id": "ext-uuid-warehouse-2", "position": 2 },
    { "extension_id": "ext-uuid-warehouse-3", "position": 3 },
    { "extension_id": "ext-uuid-shipping",    "position": 4 }
  ]
}

Field Reference

PageGroupCreate Fields

Field Type Required Default Description
name string Yes — Display name for the page group (1–255 characters)
page_number string Yes — The dial number to initiate a page to this group (1–20 characters)
description string No null Optional description of the group's purpose or location
page_mode enum No ONE_WAY Paging mode: ONE_WAY (broadcast) or TWO_WAY (intercom)
timeout int No 60 Seconds before the page automatically disconnects (1–300)
members list No [] List of PageGroupMemberCreate objects (extensions in the group)

PageGroupMemberCreate Fields

Field Type Required Default Description
extension_id UUID Yes — The extension to include in the page group
position int No 0 Sort order for display in the admin UI (0–100). Does not affect page delivery order.

Response Fields

The API response includes all create fields plus the following system-managed fields:

Field Type Description
id UUID Unique identifier for this page group
tenant_id UUID Owning tenant (set automatically from auth context)
is_active bool System-computed active state (enabled and not soft-deleted)
created_at datetime Timestamp when the group was created
updated_at datetime Timestamp of the last update
members list Full list of page group members with their IDs

Page Modes Explained

ONE_WAY (Broadcast Paging)

In one-way mode, the initiator's voice is broadcast through the speakers of all member phones simultaneously. The member phones' microphones remain off. This is a pure announcement channel — the initiator speaks, everyone listens.

  • Member phones auto-answer on speaker with microphone muted
  • The initiator speaks; members hear the announcement
  • Members cannot respond through their phones
  • The page ends when the initiator hangs up or the timeout is reached

TWO_WAY (Intercom)

In two-way mode, member phones auto-answer with both speaker and microphone active. This creates a hands-free conversation where everyone can speak and hear each other simultaneously, similar to a walkie-talkie or intercom system.

  • Member phones auto-answer on speaker with microphone active
  • All participants (initiator and members) can speak and hear each other
  • Works like a quick ad-hoc conference call without dialing a conference number
  • Best suited for small groups (2–5 phones) to avoid audio chaos
Two-Way Group Size
Two-way intercom with large groups can be difficult to use because multiple people speaking simultaneously creates crosstalk. Keep two-way groups small (2–5 members). For larger groups, use one-way paging for announcements and conferences or ring groups for multi-party conversations.

Member Management

Page group members are extensions whose phones participate in the page. When a page is initiated, the system sends a SIP INVITE with auto-answer headers to every member extension simultaneously.

  • Adding members — Include extension IDs in the members list when creating or updating the group.
  • Removing members — Update the group with a new members list that excludes the extensions you want to remove.
  • Position — The position field controls display order in the admin UI only. Pages are delivered to all members simultaneously regardless of position.
  • Busy extensions — If a member extension is on an active call when a page is initiated, that phone does not participate in the page. The page proceeds with the remaining available members.

Timeout Behavior

The timeout field sets the maximum duration of a page in seconds. When the timeout is reached, the system automatically disconnects all participants. This prevents accidental open-mic situations where the initiator forgets to hang up.

Timeout Use Case
30 Quick announcements: "Customer at front desk" or "Delivery at dock 3"
60 (default) Standard announcements with a comfortable speaking window
120 Longer announcements or two-way intercom conversations
300 Extended intercom sessions for troubleshooting or coordination

The initiator can always end the page early by hanging up. If the initiator hangs up before the timeout, all member phones disconnect immediately.

Phone Speaker Behavior

Paging relies on the SIP auto-answer mechanism to activate phone speakers without user interaction. The platform sends specific SIP headers with the page INVITE that instruct the phone to auto-answer on speaker.

Auto-Answer SIP Headers

Different phone manufacturers use different SIP headers for auto-answer. The platform sends multiple headers to maximize compatibility:

Header Phone Brand
Alert-Info: info=alert-autoanswer Polycom
Alert-Info: ;answer-after=0 Yealink, Cisco SPA
Call-Info: ;answer-after=0 Snom

Phone Configuration Requirements

Most desk phones require auto-answer to be enabled in their configuration. If pages are not working (phones ring normally instead of auto-answering), check these settings:

  • Auto-answer enabled — The phone must be configured to honor auto-answer SIP headers. This is often disabled by default.
  • Speaker volume — Ensure the phone's speaker volume is set to an audible level for the environment (warehouse, office, etc.).
  • DND interaction — Phones in Do Not Disturb mode may reject page attempts. Decide whether DND should block pages based on your environment.
Provisioning Tip
If you manage phone provisioning centrally, add auto-answer settings to your provisioning templates. This ensures all new phones are page-ready without manual configuration.

Common Use Cases

Warehouse Paging

One-way broadcast to all warehouse phones for general announcements. Use a short timeout (30–60 seconds) and include all phones on the warehouse floor. 

{
  "name": "Warehouse All-Page",
  "page_number": "700",
  "page_mode": "ONE_WAY",
  "timeout": 60,
  "members": [
    { "extension_id": "ext-dock-1",   "position": 1 },
    { "extension_id": "ext-dock-2",   "position": 2 },
    { "extension_id": "ext-floor-1",  "position": 3 },
    { "extension_id": "ext-floor-2",  "position": 4 },
    { "extension_id": "ext-shipping", "position": 5 }
  ]
}

Office Intercom

Two-way intercom between an executive assistant and office. A quick hands-free conversation without placing a formal call. 

{
  "name": "Executive Intercom",
  "page_number": "701",
  "page_mode": "TWO_WAY",
  "timeout": 120,
  "members": [
    { "extension_id": "ext-ceo-office",  "position": 1 },
    { "extension_id": "ext-assistant",   "position": 2 }
  ]
}

All-Office Page

Company-wide one-way page for emergency announcements or important notifications. Include every phone in the building. 

{
  "name": "All Office Page",
  "page_number": "799",
  "page_mode": "ONE_WAY",
  "timeout": 120,
  "description": "Emergency and company-wide announcements"
}
